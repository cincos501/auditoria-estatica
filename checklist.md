# Checklist de Auditoría Estática (IEEE 1028)

| ID | Criterio de Evaluación | Categoría | Estado |
|:---|:---|:---|:---:|
| 01 | ¿Sigue estándares de código (PascalCase)? | Estándares | ✅ |
| 02 | ¿Valida entradas de usuario para evitar excepciones? | Seguridad | ❌ |
| 03 | ¿Tiene manejo de excepciones (try-catch)? | Mantenibilidad | ❌ |
| 04 | ¿Tiene documentación técnica XML (///)? | Documentación | ❌ |
| 05 | ¿Usa tipos de datos anulables (string?) correctamente? | Buenas Prácticas | ✅ |
| 06 | ¿Frena el despliegue ante fallos de auditoría? | DevOps | ✅ |
